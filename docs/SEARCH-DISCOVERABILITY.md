# Search Discoverability

Author: Leandro Michelino | ACE | leandro.michelino@oracle.com

This repo can be made easier to find in Google, Bing, GitHub search, and AI
search summaries, but no repository can honestly guarantee a top result. The
best path is boring in the good way: clear title, clear description, relevant
topics, useful README text, real links, and regular updates.

## Practical Search Checklist

- Keep the README title explicit: Oracle Cloud Infrastructure, OCI, Terraform,
  and Ansible should appear near the top.
- Keep GitHub topics current when new blueprint families land.
- Link to this repo from blog posts, LinkedIn posts, conference decks, and
  customer-safe reference notes using descriptive anchor text.
- When creating a release, use release notes that mention the strongest new
  patterns by name.
- If GitHub Pages is enabled later, submit that page to Google Search Console
  and Bing Webmaster Tools.
- Avoid hidden keyword stuffing. It looks awkward to humans and does not age
  well with search engines.
